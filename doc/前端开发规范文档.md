# 前端规范文档

本文档概述 YouTube AI 视频翻译与发布系统前端的架构、设计原则及开发最佳实践。采用通俗易懂的语言编写，无论技术或非技术人员，都能理解 UI 搭建逻辑、技术选型原因，以及如何长期保持项目风格统一、代码可维护。

## 1. 前端架构
### 框架与依赖库
- **Next.js（React + TypeScript）**
  内置服务端渲染（SSR）、静态站点生成（SSG）和客户端水合能力。TypeScript 可提前捕获错误，同时让代码具备自文档化特性。
- **Tailwind CSS**
  实用优先的 CSS 框架，可快速、统一完成样式开发，无需为每个组件编写自定义 CSS。
- **React Query（TanStack Query）**
  统一管理数据请求、接口缓存与后台自动更新，保证界面流畅、数据实时同步。
- **身份认证**
  使用 Supabase Auth 或 NextAuth.js 保护页面和接口路由，支持账号密码登录及 GitHub 等第三方授权登录。
- **ESLint + Prettier**
  强制统一代码风格，在上线前提前发现并规避常见代码问题。

### 可扩展性、可维护性与性能支撑
- **可扩展性**
  Next.js 结合 Vercel 或 Supabase 边缘函数无服务器部署，可随流量自动弹性扩容；React Query 缓存接口响应，减少冗余请求。
- **可维护性**
  TypeScript 类型约束、ESLint 规则、Prettier 格式化保证代码风格统一；清晰的文件夹结构与可复用组件，便于新增功能迭代。
- **性能**
  服务端/静态渲染提升首屏速度与 SEO 效果；Tailwind 树摇机制精简 CSS 打包体积；配合 React Query 缓存、Next.js 图片优化，进一步加快加载速度。

## 2. 设计原则
### 核心原则
- **易用性**
  简洁易懂的业务流程。运营人员在表格查看每日视频，点击进入审核页，左右分栏编辑内容，一键完成审核通过。
- **无障碍访问**
  使用语义化 HTML 标签（如 `<button>`、`<table>`）、规范 ARIA 属性、键盘聚焦样式、合理的色彩对比度。
- **响应式适配**
  采用移动优先设计，基于 Tailwind 响应式工具类，仪表盘、表格、编辑面板可完美适配桌面、平板、手机设备。
- **清晰性**
  内容优先展示；原文与翻译分栏明确标注，配备自动保存提示与状态标签。

### 原则落地规范
- 按钮、链接标签语义清晰，具备悬浮和聚焦交互状态。
- 表单包含标签、占位提示及行内错误提示。
- 状态标签结合颜色、图标与文字标识：待审核、已通过、已驳回。
- 侧边栏在小屏自动折叠，主体内容垂直自适应堆叠。

## 3. 样式与主题规范
### 样式开发方式
- **Tailwind 原子类优先**
  直接使用 `px-4`、`bg-indigo-600`、`text-gray-900` 等组合样式，无需编写冗长 CSS 选择器。
- **无需预处理器**
  Tailwind 内置变体、响应式前缀、自定义主题能力，无需依赖 SASS / LESS。
- **CSS 按需精简**
  Tailwind 编译时自动移除未使用样式，缩减打包体积。

### 主题模式
- **浅色 / 深色模式**
  通过 Tailwind `dark:` 变体控制；用户主题偏好存入 localStorage 与 React 上下文，切换即时生效。
- **CSS 变量定制**
  弹窗、玻璃拟态卡片等自定义组件，在 `:root` 定义 `--card-bg`、`--text-color` 等变量，深色模式下覆盖重写。

### 视觉风格
整体采用现代扁平化设计，弹窗与卡片使用轻微玻璃拟态效果，轻量化营造视觉层次感。

- **标题与正文**：正文使用 `font-normal`，标题使用 `font-medium` 区分层级。

## 4. 组件结构
### 目录结构

```
src/
├─ pages/            # Next.js page routes
│  ├─ index.tsx      # Dashboard
│  ├─ videos/        # /videos/[id].tsx (Review page)
│  └─ settings.tsx   # User settings
├─ components/       # Reusable UI components
│  ├─ Button/        # Button.tsx + Button.test.tsx
│  ├─ Table/         # VideoListTable.tsx
│  └─ Modal/         # Modal.tsx
├─ layouts/          # App shell (sidebar + header)
├─ hooks/            # Custom React hooks (useAuth, useVideos)
├─ context/          # React Context providers (Theme, Auth)
├─ services/         # API calls (supabaseClient, youtubeApi)
├─ styles/           # Global styles, Tailwind config
└─ utils/            # Helpers (date formatting, error handling)
```


### 组件复用与分层
- **原子基础组件**：按钮、输入框、状态标签等最小粒度组件，独立目录维护。
- **业务功能组件**：视频列表、审核编辑面板、工具栏，由基础组件组合封装。
- **价值**：结构清晰、职责单一、便于单元测试与长期维护迭代。

## 5. 状态管理
### 技术选型与模式
- **React Query**：统一管理服务端状态，处理视频列表、详情、审核状态，变更后自动更新缓存。
- **Context API**：管理全局 UI 状态，如主题、登录用户信息。
- **局部状态**：组件内部独立状态使用 `useState` / `useReducer`。

### 数据流逻辑
1. **数据请求**：通过 `useQuery` 获取视频列表，渲染仪表盘。
2. **接口变更**：审核操作使用 `useMutation`，执行后失效缓存、自动刷新列表。
3. **全局上下文**：登录上下文向所有页面提供用户信息与退出登录方法。

整体保证数据一致、避免属性层层透传，界面响应更流畅。

## 6. 路由与导航
### Next.js 路由规则
- **页面目录路由**：`src/pages` 目录文件直接映射访问路径
  - `/` 仪表盘首页
  - `/videos/[id]` 视频详情审核页
  - `/settings` 用户设置页
- **动态路由**：通过 `[id].tsx` 动态匹配不同视频 ID。

### 客户端导航
- 使用 `next/link` 做内部跳转，支持预加载，页面切换更快。
- 侧边栏通过 `useRouter().pathname` 匹配路由，自动高亮当前菜单。

### 导航结构
- **常驻侧边栏**：仪表盘、设置菜单固定展示，移动端可折叠。
- **顶部导航**：网站 Logo、用户头像菜单、退出登录。
- **面包屑（可选）**：详情页展示路径层级，提升定位感。

## 7. 性能优化
### 核心优化策略
- **服务端渲染 SSR / 静态生成 SSG**：仪表盘使用 SSR 保证数据实时；详情页采用 SSG 预渲染加兜底降级。
- **代码分割与懒加载**：通过 `next/dynamic` 懒加载富文本预览等重型组件。
- **图片优化**：使用 `next/image` 组件自动完成图片缩放、格式适配与缓存。
- **缓存策略**：React Query 缓存复用，回访页面减少重复请求。
- **打包分析**：通过打包体积分析工具，严控依赖大小，优化首屏加载。

## 8. 测试与质量保障
### 测试体系
- **单元测试**：Jest + React 测试库，覆盖基础组件与自定义钩子。
- **集成测试**：使用 MSW 模拟接口，测试页面完整交互逻辑。
- **端到端 E2E 测试**：Cypress / Playwright 模拟真实用户操作：登录、审核视频、编辑文本、提交审批。
- **无障碍测试**：接入 axe 工具，自动检测无障碍合规问题。

### 工程化质量工具
- **ESLint**：代码提交强制校验规范。
- **Prettier**：保存自动格式化，统一团队编码风格。
- **CI/CD 流水线**：GitHub Actions 自动执行代码检查、单元测试、E2E 测试，校验通过后部署至 Vercel。

## 9. 总结
本文档规范了基于 Next.js + TypeScript + Tailwind CSS 搭建的前端整体架构，结合 React Query 数据管理与 Supabase 身份认证，打造高性能、可扩展、易维护的后台管理系统。

项目严格遵循易用性、无障碍、响应式、视觉清晰四大设计原则，配合规范的组件分层、状态管理、路由设计、性能优化与完整测试体系，支撑团队持续迭代新功能、快速修复问题。

整套架构可完整支撑 YouTube 视频抓取、AI 翻译审核、自动发布全业务流程，同时为运营人员提供流畅稳定的使用体验。
