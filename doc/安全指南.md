# 安全指南：YouTube AI 视频翻译与发布系统

本文档面向开发团队，对每天从 YouTube 抓取 AI 技术热点视频、提取内容、翻译、存储到 Supabase 并发布到知乎的系统，提供全流程的安全设计和实现规范。

---

## 1. 界面风格与品牌要求

- **整体风格**：简约、科技感
  - 主色调：深蓝或黑白
  - 辅助色：亮蓝、青色用于高亮、按钮、链接
- **字体**：
  - 正文字体：思源宋体或 Noto Sans 简体
  - 标题：思源黑体或 Inter
- **品牌一致性**：
  - Logo 和应用名称放置在导航栏左上角
  - 所有页面统一 Header/Footer
- **无障碍**：
  - 按钮和表单元素满足 WCAG 对比度要求
  - 支持键盘操作与屏幕阅读器

## 2. 多用户与权限管理（RBAC）

- **角色设计**：
  - `admin`（管理员）：
    - 抓取任务配置、翻译审核、发布到知乎
  - `editor`（译者）：
    - 查看待翻译列表、编辑翻译内容、保存草稿
  - `viewer`（只读）：
    - 查看视频列表与翻译结果
- **授权流程**：
  - 登录后根据 JWT 或 Supabase Auth Token 判断角色
  - 服务端每个 API Route 强制检查 `role` 字段
- **最小权限**：仅授予角色所需接口访问权限
- **会话管理**：
  - HttpOnly+Secure+SameSite=Strict Cookie 存储 Session
  - Idle 超时 30 分钟、最长会话 8 小时
  - 强制登出、刷新令牌时更新 Cookie

## 3. Next.js 前端页面与核心功能

1. **登录/登出页**
   - Email/密码 + OAuth（GitHub）
   - 表单校验（最小 8 位含大小写数字符号）
2. **视频列表页**
   - 按状态（待翻译、待审核、已发布）筛选
   - 搜索与分页
3. **翻译详情页**
   - 左：原文转录；右：AI 翻译，可编辑
   - 操作：保存草稿、提交审核、驳回并备注
4. **审核管理页**（仅 admin）
   - 列表查看所有待审核翻译
   - 批量通过/驳回
   - 审核日志查看
5. **发布与导出页**
   - 生成 Markdown 并下载
   - 发布到知乎按钮（触发后端调用）
6. **设置页**
   - 修改密码、通知偏好
   - 授权第三方服务（如 OpenAI、YouTube API）

所有输入与编辑内容需进行 XSS 过滤与输出编码。

## 4. 定时任务部署方案

- **可选方案**：
  - Vercel Cron Jobs（Serverless Functions）
  - Supabase Scheduled Edge Functions
- **执行权限**：仅使用 Service Role Key（仅在后端环境）
- **安全配置**：
  - 环境变量存储在平台中，不写入代码
  - 最多重试 3 次，失败发送告警邮件/Slack
  - 防止并发任务互斥锁（Redis 分布式锁）

## 5. 审核流程界面与操作规范

- **待审核列表**：
  - 展示视频 Title、缩略图、上传时间、选中复选框
- **审核操作**：
  - 单个/批量通过或驳回
  - 驳回时强制填写原因
- **日志记录**：
  - 存储操作人、时间、动作、备注
  - 审核历史可追溯
- **前后端校验**：
  - 防止 CSRF（同步令牌模式）
  - 操作幂等性：同一记录只能审核一次

## 6. 视频内容获取与转录安全

- **数据来源**：YouTube Captions API，若无自带字幕则使用 OpenAI Whisper
- **输入校验**：
  - 视频 ID 格式校验（^[a-zA-Z0-9_-]{11}$）
  - 限制单次传输文件大小
- **文件上传**：
  - 暂存音频文件于后端临时目录，权限 600
  - 处理完毕后及时删除
- **网络传输**：
  - TLS 1.2+ 全链路加密
- **隐私合规**：
  - 不存储个人敏感信息，仅保留转录文本

## 7. 翻译服务方案与安全

- **可选方案**：OpenAI GPT-4、Google Translate API、开源模型（需自建）
- **准确度 vs 成本**：
  - GPT-4：高质量，单次请求成本较高
  - Google：中等质量，稳定性好
  - 开源模型：成本最低，但需运维 GPU 集群
- **速率限制**：
  - 每分钟限制调用次数，防止滥用
- **凭证管理**：
  - API Key 存放于环境变量
  - 权限最小化，仅具备翻译能力

## 8. Supabase 数据表设计与安全

- **`videos` 表字段**：
  - `id` UUID 主键
  - `youtube_id` TEXT
  - `title` TEXT
  - `thumbnail_url` TEXT
  - `original_transcript` TEXT
  - `translated_text` TEXT
  - `status` ENUM('pending','reviewing','approved','rejected','published')
  - `author_id` UUID 引用 `users` 表
  - `ingest_time` TIMESTAMP
  - `review_time` TIMESTAMP
  - `publish_time` TIMESTAMP
- **行级安全（RLS）**：
  - `admin` 可读写所有行
  - `editor` 仅能操作自己 `author_id` 的行
  - `viewer` 仅能 SELECT 状态为 `approved` 或 `published` 的行
- **索引**：
  - `status`、`ingest_time` 以加速查询
- **加密**：
  - 数据库连接使用 SSL
  - 敏感日志脱敏

## 9. 发布至知乎的安全方案

- **方案对比**：
  - 调用官方公开 API（优先）
  - Puppeteer 模拟用户操作（备用），注意处理验证码
- **认证方式**：
  - OAuth2：服务端完成授权 Code→Token，存储 Refresh Token
  - Cookie 模拟登录：仅限模拟方案，需定期人工更新
- **权限管理**：
  - Token 存放在后端机密存储
  - 不在前端暴露任何凭证

## 10. 热点 AI 技术视频定义与抓取策略

- **来源**：YouTube Data API v3
- **筛选条件**：
  - `chart=mostPopular` & `videoCategoryId=28`（Science & Tech）
  - `regionCode` 可配置（如 CN/US）
- **关键词补充**：可根据设定关键字（AI、Deep Learning、Machine Learning）进一步过滤标题
- **数量限制**：每日抓取上限 20 条
- **缓存机制**：接口结果缓存 15 分钟，防止重复拉取耗尽配额

---  

## 总体安全最佳实践

- **安全凭证管理**：
  - 使用专用 Secrets Manager（或 Vercel/Supabase 内置）
  - 环境变量声明式：不在代码库中出现明文 Key
- **输入验证与输出编码**：
  - 后端全面校验所有外部输入
  - 前端对展示文本做 HTML 转义
- **错误与日志**：
  - 不泄露堆栈、内部路径、数据库信息
  - 审计日志记录用户敏感操作
- **通信加密**：
  - HTTP 全面启用 HTTPS
  - 禁用 TLS 1.0/1.1、SSLv3
- **依赖管理**：
  - 使用 `package-lock.json` / `yarn.lock`
  - 定期扫描 CVE，及时升级

遵循以上安全设计，能够确保系统在高可用的同时，满足数据保护、访问控制和审计可追溯等关键安全需求。
