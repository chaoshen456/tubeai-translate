## 1. 前端技术栈
运营审核后台供编辑人员审核、确认译文，采用现代化 Web 技术搭建，保证界面加载快、响应流畅、易于维护。

- **Next.js（React + TypeScript）**
  - 基于 React 生态，自带服务端渲染能力，页面加载速度快。
  - TypeScript 提供类型约束，提前捕获代码错误，提升项目健壮性。

- **Tailwind CSS**
  - 原子化样式框架，无需为每个组件编写自定义 CSS，快速搭建风格统一、简洁干净的界面。

- **Supabase Auth / NextAuth.js**
  - 支持邮箱密码、GitHub 第三方授权登录与注册。
  - 无缝集成 Next.js，可对页面和接口路由做登录权限拦截。

- **React Query（TanStack Query）**（可选推荐）
  - 简化视频列表、译文草稿等数据请求，自带缓存机制，保证页面交互流畅。

- **ESLint + Prettier**
  - 统一代码规范风格，上线前提前排查常见代码问题。

- **VSCode + Supabase 插件**
  - 提供友好的开发环境，方便代码编写、调试、数据库操作。

### 对用户体验的提升
- 依托 Next.js 渲染策略，页面秒开、切换顺滑。
- Tailwind 原生适配移动端，界面易用且符合无障碍规范。
- React Query 后台静默更新数据，支持编辑自动保存等实时反馈。

## 2. 后端技术栈
后端负责视频抓取、语音转写、AI 翻译、数据存储，基于无服务器函数 + 托管数据库构建。

- **YouTube Data API v3 & 字幕 API**
  - 通过 `videos.list` 获取热门 AI 分类每日榜单视频。
  - 优先获取官方字幕，无字幕时自动降级走语音转写。

- **OpenAI Whisper（语音转文字）**
  - 当 YouTube 无字幕或字幕质量差时，将视频音频转为文本文稿。

- **OpenAI GPT-4**
  - 对分段字幕进行高质量、具备上下文理解的中文翻译。

- **Next.js 接口路由 / 独立 Node.js 服务**
  - 提供业务接口，触发自动化流水线、查询数据、更新审核状态。

- **Supabase（PostgreSQL + 边缘函数）**
  - 数据库存储视频元信息、原始字幕、译文、审核状态、时间戳等。
  - 边缘函数基于 JS/TS 编写轻量无服务器代码，可运行定时任务与按需查询。

### 核心业务流转
1. 定时任务调用 YouTube 接口 → 获取视频元信息 + 字幕。
2. 若无官方字幕 → 音频送入 Whisper 转文字。
3. 字幕分段拆分 → 调用 GPT-4 逐段翻译。
4. 所有数据存入 Supabase `videos` 表，初始状态为**待审核**。
5. 前端审核页面可更新状态为**通过/驳回**，并记录审核时间。

## 3. 基础设施与部署
整套架构保障系统稳定、迭代便捷、每日任务自动运行。

- **Vercel 托管**
  - 托管 Next.js 前端与接口路由。
  - 原生支持环境变量，安全存放各类密钥与数据库地址。
  - 提供 Cron 定时任务，支撑每日自动化流水线。

- **Supabase 定时边缘函数**（Vercel 定时任务替代方案）
  - 可独立调度执行视频抓取流水线，业务逻辑贴近数据库部署。

- **GitHub & GitHub Actions**
  - 代码版本管理，分支与 PR 管控迭代变更。
  - 配置 CI/CD 流水线，合并到 `main` 分支自动代码检查、测试并部署到 Vercel。

- **环境变量**
  - 所有密钥（YouTube、OpenAI、数据库地址）统一配置在 Vercel / Supabase 后台，不硬编码进代码。

- **版本管理与回滚**
  - Vercel、Supabase 保留历史部署版本，出现故障可快速一键回滚。

### 架构收益
- **弹性扩容**：无服务器函数自动适配业务负载。
- **高可用**：每日任务无需人工值守，自带重试与告警机制。
- **易维护**：基于 GitHub + Vercel 统一发布，版本迭代可控可预期。

## 5. 安全与性能设计考量
从数据安全、访问权限、接口性能多维度做优化防护。

### 安全设计
- **基于角色权限控制 RBAC**（依托 Supabase Auth）
  仅 `admin` 管理员角色可操作审核通过/驳回。

- **Supabase 行级安全 RLS**
  严格控制数据行权限，用户只能查看授权范围内数据。

- **全站 HTTPS**
  所有接口请求均采用加密安全连接。

- **密钥管理**
  密钥统一存环境变量，代码中不泄露敏感凭证。

### 性能优化
- **批量请求 + 缓存**
  单次 YouTube 接口最多拉取 50 条视频；热门列表缓存 15–30 分钟，减少第三方调用频次。

- **并发控制**
  翻译分段并行处理，控制单段翻译耗时在 5 秒内。

- **指数退避重试**
  YouTube / OpenAI 接口失败自动重试，多次失败触发告警。

- **数据库索引**
  对 `status` 状态、`ingest_time` 抓取时间建立索引，后台列表查询极速响应。

## 2. Backend Technologies

The backend handles video fetching, transcription, translation, and data storage. It’s built using serverless functions and a managed database.

- **YouTube Data API v3 & Captions API**
  • Fetches daily trending AI-tech videos via `videos.list(chart='mostPopular', videoCategoryId='28')`.
  • Retrieves available captions; if none exist, we fallback to transcription.
- **OpenAI Whisper (speech-to-text)**
  • Converts raw video audio to text when YouTube captions are absent or low quality.
- **OpenAI GPT-4**
  • Translates transcript segments into high-quality, context-aware Chinese text.
- **Next.js API Routes or Standalone Node.js Service**
  • Houses endpoints to trigger the pipeline, fetch data, and update review status.
- **Supabase (PostgreSQL + Edge Functions)**
  • Database tables store video metadata, original transcript, translated text, status flags, timestamps, and author info.
  • Edge Functions provide lightweight serverless code (written in JavaScript/TypeScript) to run scheduled tasks or on-demand queries.

Key interactions:
1. Scheduled job calls YouTube API → metadata + captions.
2. If captions missing → send audio file to Whisper.
3. Split transcript into chunks → call GPT-4 for translation.
4. Save everything in Supabase’s `videos` table with status `Pending Review`.
5. Review UI updates status to `Approved` or `Rejected` and records review timestamps.

## 3. Infrastructure and Deployment

Our infrastructure ensures reliability, easy updates, and automatic daily runs.

- **Vercel for Hosting**
  • Hosts the Next.js frontend and API routes.
  • Built-in support for Environment Variables to store API keys and database credentials securely.
  • Provides Cron Jobs capability (Serverless scheduled functions) for daily pipeline runs.
- **Supabase Scheduled Edge Functions** (alternative to Vercel Cron)
  • Can also run our ingestion pipeline on a schedule, keeping logic close to the database.
- **GitHub & GitHub Actions**
  • Source control of all code, with branches and pull requests for changes.
  • CI/CD pipeline runs linting, tests, and deploys to Vercel on every merge to `main`.
- **Environment Variables**
  • All secrets (YouTube API key, OpenAI keys, database URLs) live in Vercel/Supabase dashboards, never in code.
- **Versioning and Rollbacks**
  • Vercel and Supabase maintain previous deployments, enabling quick rollbacks if something breaks.

These choices give us:
- **Scalability**: Serverless functions auto-scale to handle workload.
- **Reliability**: Daily jobs run without manual intervention, with retry logic and alerts.
- **Maintainability**: Centralized deployments via GitHub and Vercel make updates predictable.

## 5. Security and Performance Considerations

We implemented measures to protect data and keep the application fast.

Security
- **Role-Based Access Control (RBAC)** via Supabase Auth
  • Only users with the `admin` role can approve or reject translations.
- **Row-Level Security (RLS)** in Supabase
  • Ensures users see only what they’re permitted.
- **HTTPS Everywhere**
  • All API calls use secure connections.
- **Secret Management**
  • Environment variables in Vercel/Supabase—no secrets in code.

Performance
- **Batching & Caching**
  • Fetch up to 50 video IDs per YouTube API call.
  • Cache trending results for 15–30 minutes to reduce external calls.
- **Concurrency Control**
  • Parallelize transcript translation chunks to meet the <5-second target per chunk.
- **Retries with Exponential Back-off**
  • Automatic retries on YouTube/OpenAI failures, then alert if all attempts fail.
- **Database Indexing**
  • Index on `status` and `ingest_time` fields for quick dashboard queries.
