# 项目需求文档 (PRD)

## 1. 项目概述
本项目是一套每日自动运行的全流程自动化管线，用于从YouTube抓取最新的AI技术热点视频、提取视频中的语音内容、将其翻译为中文，并将原始转录文本和翻译文本均存储至Supabase数据库。经人工审核员确认翻译内容无误后，系统会生成可直接发布的Markdown格式稿件，或自动将内容推送至知乎账号。这为内容团队节省了数小时的手工操作时间，并确保中文用户能及时获取最新的AI资讯。

核心目标：
- 实现从视频发现到数据存储的端到端自动化
- 借助顶尖AI模型完成高质量、贴合语境的中文翻译
- 开发简洁的Next.js管理后台，支持翻译内容的审核、编辑与确认
- 实现向知乎的无缝发布流程（v1版本支持手动导出，v2版本通过API或Puppeteer实现自动发布）

成功衡量标准：
1. 每日管线运行零故障
2. 每段视频内容的翻译延迟低于5秒
3. 审核界面简洁易用，编辑可在30分钟内处理20条视频
4. 审核通过的稿件可在数分钟内成功发布至知乎

## 2. 功能范围界定

### 纳入范围（1.0版本）
- 定时任务（Cron）：通过YouTube Data API v3抓取热门AI技术视频
- 转录提取：使用YouTube字幕API；若无字幕则降级至OpenAI Whisper
- 翻译服务：将清洗后的转录文本片段发送至OpenAI GPT-4接口完成中文化翻译
- 数据存储：在Supabase中存储视频元数据、原始转录文本、翻译文本、状态、时间戳和缩略图URL
- Next.js管理后台：
  - 登录认证、视频列表展示、原始/翻译文本对比查看
  - 翻译文本在线编辑、草稿保存、审核通过/驳回操作
- 导出功能：将审核通过的内容生成Markdown（.md）文件供手动下载
- 环境变量管理：统一管理所有API密钥和数据库凭证

### 暂不纳入（规划至后续版本）
- 非每日调度的实时/按需处理能力
- 中文以外的其他语言支持
- 自动化唇形同步配音或视频编辑功能
- 与知乎私有/未公开API的深度集成（v2仅支持基于浏览器的自动化）
- 高级AI功能（如情感分析、多媒体摘要、场景检测）

## 3. 用户流程
编辑登录系统后，会进入仪表盘页面，展示当日及往期抓取的所有视频。每条列表项包含视频标题、缩略图、YouTube链接、抓取时间戳，以及状态标签：「待翻译」「待审核」或「已发布」。编辑点击任意行可打开详情页，页面分为左右双面板：左侧展示原始转录文本，右侧展示AI翻译的中文文本。编辑可在线编辑中文文本、保存草稿，最终点击「审核通过」或「审核驳回」。

审核通过后，系统将该记录标记为「待发布」。编辑可点击「生成Markdown」按钮下载预格式化的.md文件，包含标题、缩略图嵌入、带时间戳的翻译内容和YouTube链接。在2.0版本中，该步骤将被「发布至知乎」按钮替代/增强，触发API或Puppeteer脚本直接发布至关联的知乎账号，并自动完成标签和分类配置。

## 4. 核心功能
- **调度模块**：通过Vercel Cron或Supabase定时边缘函数每日运行
- **YouTube抓取器**：调用`videos.list(chart='mostPopular', videoCategoryId='28')`（AI分类），支持地区编码配置
- **转录提取器**：优先使用YouTube字幕API；无字幕时通过OpenAI Whisper批量转录
- **翻译服务**：将清洗后的转录文本片段发送至OpenAI GPT-4端点完成中文化翻译
- **Supabase数据层**：`videos`表字段包括：`id`、`youtube_id`、`title`、`thumbnail_url`、`original_transcript`、`translated_text`、`status`、`ingest_time`、`review_time`、`publish_time`
- **Next.js管理后台**：
  - 认证：仅面向管理员的邮箱/密码或OAuth（如GitHub）登录
  - 视频列表页：按状态筛选、按标题搜索
  - 视频详情页：双面板对比编辑、草稿保存、版本历史、审核操作

- **导出/发布模块**：
  - Markdown导出：模板包含前置元信息（标题、日期、标签）、带时间戳的正文内容
  - （v2）知乎发布：OAuth2集成或Puppeteer脚本自动填充知乎发布编辑器并提交

## 5. 技术栈与工具
- **前端**：Next.js（React）、TypeScript、Tailwind CSS
- **后端/API路由**：Next.js API路由（Node.js）或独立Node.js服务
- **数据库**：Supabase（PostgreSQL）
- **调度器**：Vercel Cron Jobs 或 Supabase定时函数
- **YouTube Data API v3**：获取视频元数据和字幕
- **OpenAI API**：
  - Whisper：备用语音转文字方案
  - GPT-4：翻译核心能力
- **知乎集成**：
  - v1：手动Markdown导出
  - v2：Puppeteer浏览器自动化或知乎开放平台（如有）
- **认证**：Supabase Auth 或 NextAuth.js
- **部署环境**：Vercel（托管）、GitHub Actions（CI/CD）
- **开发工具**：VSCode、Supabase VSCode扩展、ESLint、Prettier

## 6. 非功能性需求
- **性能**：
  - 每日全流程（抓取→转录→翻译→存储）在20条视频规模下需5分钟内完成
  - 每1分钟转录文本片段的翻译API响应时间≤5秒
- **安全**：
  - 所有API密钥和机密信息存储在环境变量中，禁止硬编码
  - 所有API调用强制使用HTTPS
  - 基于角色的访问控制：仅管理员可查看/修改翻译内容
- **可靠性**：
  - 对临时API故障（YouTube/OpenAI）自动重试（指数退避策略）
  - 管线错误超过阈值时触发Slack/邮件告警
- **易用性**：
  - 适配移动端的管理后台界面
  - 在线编辑内容每30秒自动保存
- **合规性**：
  - 遵守YouTube API配额限制：热门视频列表缓存15分钟
  - 数据保留策略：清理超过30天的未审核记录

## 7. 约束与假设
- **YouTube API**：API密钥具备足够配额；地区编码默认`US`且可配置
- **Supabase**：免费版限额（带宽、存储）满足初期规模需求
- **翻译质量**：GPT-4 API可用且在速率限制内；假设平均转录文本片段≤2000令牌
- **知乎集成**：官方发布API可能不存在；Puppeteer自动化需模拟人工操作并处理验证码
- **调度能力**：平台（Vercel/Supabase）支持免费Cron任务
- **人工审核**：每日有编辑负责审核/驳回翻译内容

## 8. 已知问题与潜在风险
- **字幕缺失/低质量**：大量AI技术视频无字幕或自动字幕质量差；缓解方案：降级至Whisper并限流音频上传
- **API速率限制**：超出YouTube/OpenAI配额；缓解方案：结果缓存、批量请求、用量监控
- **翻译语境丢失**：GPT可能误译专业术语；缓解方案：在审核界面增加术语表/手动覆盖功能
- **知乎反爬机制**：Puppeteer脚本可能因知乎DOM更新/验证码失效；缓解方案：降级至手动导出，持续维护脚本
- **数据一致性**：部分失败（如元数据保存成功但翻译失败）导致孤立记录；缓解方案：事务包裹或基于状态机的重试逻辑
