# 后端架构文档

本文档以简洁清晰的语言说明 YouTube AI 视频自动翻译与发布系统的后端配置，涵盖架构、数据库设计、API、部署、基础设施、安全与维护。

## 1. 后端架构
系统采用无服务器、模块化设计，每个模块独立负责一项功能，可单独扩展。

- **定时自动化流水线（Cron Job）**
  - 每天运行一次，从 YouTube 抓取热门 AI 科技类视频。
  - 基于 Vercel Cron Job 或 Supabase 定时边缘函数实现。
  - 执行步骤：
    1. 调用 YouTube Data API v3，获取 AI 类别下的热门视频。
    2. 尝试通过 YouTube 字幕接口获取字幕；若无字幕，则调用 OpenAI Whisper 进行语音转文字。
    3. 将字幕分段，发送到 OpenAI GPT-4 生成中文翻译。
    4. 将视频元数据、原文、译文、状态保存到 Supabase。

- **API 层**
  - 基于 Next.js API 路由（Node.js）构建，也可使用独立 Node 服务。
  - 处理数据查询、翻译编辑、审核操作、发布触发。

- **管理后台**
  - Next.js 前端调用 API 路由。
  - 提供视频列表、字幕查看/编辑、审核通过/驳回页面。

架构优势：
- **可扩展性**：无服务器函数自动随负载扩容。
- **可维护性**：抓取、提取、翻译、数据库、前端分模块存放。
- **性能**：热门视频列表缓存 15 分钟，API 并行调用，每日任务可在 5 分钟内完成。

## 2. 数据库管理
系统使用 **Supabase（托管 PostgreSQL）** 存储所有数据。
Supabase 提供：
- **SQL 数据库**：可靠、支持事务。
- **认证与行级安全（RLS）**：内置用户账号与细粒度权限规则。
- **边缘函数**：靠近数据库的无服务器代码。
- **备份与监控**：每日自动备份与使用面板。

数据处理规范：
- 热门视频列表使用内存或短期表缓存。
- 清理超过 30 天未审核的记录。
- 多步骤操作（抓取→翻译→存储）使用状态机确保一致性，避免无效数据。

## 3. 数据库结构
### 结构概览
系统包含三张主要表：
1. **videos**：存储每个 YouTube 视频信息与翻译状态。
2. **users**：存储管理员信息与角色。
3. **user_preferences**：用户偏好设置，如通知开关。

### SQL 表结构
```sql
-- 1. 视频表
CREATE TABLE public.videos (
  id              SERIAL PRIMARY KEY,
  youtube_id      TEXT UNIQUE NOT NULL,
  title           TEXT NOT NULL,
  thumbnail_url   TEXT,
  original_text   TEXT,         -- 完整字幕
  translated_text TEXT,         -- AI 生成中文文本
  status          TEXT NOT NULL,
     -- 枚举：Pending Translation, Pending Review, Approved, Rejected, Ready to Publish, Published
  ingest_time     TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  review_time     TIMESTAMP WITH TIME ZONE,
  publish_time    TIMESTAMP WITH TIME ZONE,
  rejection_note  TEXT
);

-- 2. 用户表（由 Supabase Auth 管理）
CREATE TABLE public.users (
  id        UUID PRIMARY KEY,
  email     TEXT UNIQUE NOT NULL,
  role      TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 3. 用户偏好表
CREATE TABLE public.user_preferences (
  user_id        UUID REFERENCES public.users(id) ON DELETE CASCADE,
  notify_email   BOOLEAN DEFAULT TRUE,
  notify_slack   BOOLEAN DEFAULT FALSE,
  PRIMARY KEY (user_id)
);

-- 索引，加速查询
CREATE INDEX idx_videos_status ON public.videos(status);
CREATE INDEX idx_videos_ingest_time ON public.videos(ingest_time);
```

## 4. API 设计与接口

系统基于 Next.js 接口路由采用 RESTful 设计风格，核心接口如下：

- **GET /api/videos**
  返回视频列表，支持按状态、日期筛选。

- **GET /api/videos/:id**
  返回单个视频完整详情，包含字幕文稿。

- **POST /api/videos/:id/save**
  保存编辑后的翻译译文为草稿，不改变审核状态。

- **POST /api/videos/:id/approve**
  将视频标记为 `Ready to Publish（待发布）`，并记录审核时间。

- **POST /api/videos/:id/reject**
  将视频标记为 `Rejected（已驳回）`，可选择性保存驳回备注。

- **POST /api/videos/:id/publish**
  - 版本1：生成并返回 Markdown 文件。
  - 版本2：通过 Puppeteer 或知乎开放接口，触发自动发布到知乎。

- **POST /api/scheduler/run**
  手动触发每日自动化流水线任务（供定时任务调用）。

- **认证相关接口**（由 Supabase Auth 接管）
  包含 `/api/auth/signup`、`/api/auth/signin` 以及密码重置流程。

## 5. 托管部署方案

- **Vercel**
  - 托管 Next.js 全应用（前端 + 接口路由）。
  - 原生支持环境变量配置（YouTube 密钥、OpenAI 密钥、Supabase 地址等）。
  - 提供定时任务能力，用于运行无服务器定时函数。

- **Supabase**
  - 提供 PostgreSQL 数据库与账号认证服务。
  - 可部署定时边缘函数，作为 Vercel 定时任务的替代方案。

方案优势：
- **高可用性**：托管级服务，具备高可用 SLA 保障。
- **弹性扩容**：无服务器接口与托管数据库可随业务流量自动扩容。
- **成本可控**：按量付费，无闲置服务器资源开销。

## 6. 基础设施组件

- **负载均衡与 CDN**
  依托 Vercel 全球边缘网络，分发静态资源并缓存接口响应。

- **缓存策略**
  热门视频列表采用内存缓存或 Supabase 临时表缓存，缓存时长 15 分钟。

- **任务调度器**
  通过 Vercel 定时任务 或 Supabase 定时函数，执行视频抓取流水线。

- **函数执行**
  基于无服务器函数，并行批量处理视频抓取、语音转写、AI 翻译。

- **CI/CD 流水线**
  接入 GitHub Actions，每次合并到 `main` 分支自动执行代码校验、测试与部署。

## 7. 安全防护措施

- **认证与授权**
  - 由 Supabase Auth 管理账号密码登录与第三方 OAuth 登录。
  - 基于角色权限控制，仅管理员可修改视频审核状态。
  - PostgreSQL 行级安全策略（RLS），防止越权访问数据。

- **数据加密**
  - 全站启用 HTTPS（Vercel 免费 SSL）。
  - Supabase 数据库静态存储加密。

- **密钥管理**
  所有接口密钥、凭证均配置在 Vercel / Supabase 环境变量中，不硬编码到代码。

- **网络安全**
  边缘函数与数据库处于私有网络环境，仅对外暴露公开 API 接口。

## 8. 监控与日常维护

- **日志与告警**
  - 应用日志由 Vercel 收集，错误异常接入 Sentry 上报。
  - 流水线任务失败自动指数退避重试；多次重试仍失败，自动推送 Slack 或邮件告警。

- **性能指标**
  - 通过 Supabase 面板监控数据库查询耗时、连接数等指标。
  - 自定义业务指标（如每日流水线执行时长）同步至监控平台或 Slack。

- **备份与回滚**
  - Supabase 自动执行数据库每日备份。
  - Vercel 保留历史部署版本，支持快速版本回滚。

- **日常维护**
  - 借助 Dependabot 定期更新项目依赖。
  - 每月复盘接口配额使用量与账单成本。

## 9. 总结与后端整体概述
本系统后端采用 Next.js + Supabase + OpenAI 构建**无服务器模块化架构**。
系统每日自动完成视频发现、语音转写、AI 翻译与数据入库；运营人员通过简易管理后台完成译文查看、编辑与审核。
审核通过的内容可导出 Markdown 或一键发布至知乎。

依托 Vercel 与 Supabase 托管服务、自动化监控体系以及完善的安全规范，整套系统稳定可靠、成本低廉且易于维护，可满足每日为中文社区推送最新 AI 科技资讯的业务目标。
